# ClipFrame(画面部分切り取り君II)仕様書

- 版: 0.1(初版ドラフト)
- 日付: 2026-07-19
- 状態: 仕様検討中

## 1. 背景と目的

Microsoft Teams の画面共有は「画面全体」または「ウィンドウ」単位のみで、画面の任意領域を切り出して共有できない(Zoom には「画面の部分」共有がある)。

ClipFrame は、ユーザーが指定した画面領域をリアルタイムに映し続ける通常ウィンドウ(ミラーウィンドウ)を提供する。Teams からはそれが普通の「ウィンドウ」として見えるため、ウィンドウ共有機能を使って実質的な領域共有を実現する。

本アプリの機能はあくまで「領域を指定し、その内容を Window として認識可能にする」ことに限定する。共有そのものは Teams 等の会議アプリに委ねる。

## 2. 名称

| 項目 | 名称 |
|---|---|
| 製品名 | ClipFrame |
| 和名サブタイトル | 画面部分切り取り君II |
| 実行ファイル名 | ClipFrame.exe |
| ミラーウィンドウのタイトル | ClipFrame View |

命名意図: 「clip する frame = 切り取る枠」。枠がこのアプリの実体である。既存ツール RegionToShare(同種 OSS)や FrameClipper(動画フレーム抽出 Web ツール)との混同を避けた。

## 3. 全体構成

2 ウィンドウ構成とする。

1. **枠オーバーレイ**(共有領域の可視化と操作 UI)— キャプチャから除外され、共有映像には映らない
2. **ミラーウィンドウ**(指定領域の内容を表示する通常ウィンドウ)— Teams で共有される対象

```
デスクトップ
┌─────────────────────────────┐
│  ┌──[⠿ タグ]──────────┐     │
│  │                     │     │   枠オーバーレイ(除外)
│  │   共有される領域     │     │   ← 枠線は領域の外側
│  │   (素通し・操作可)  │     │
│  └─────────────────────┘     │
│                 ┌──────────┐ │
│                 │ClipFrame │ │   ミラーウィンドウ(通常)
│                 │  View    │ │   ← Teams でこれを共有
│                 └──────────┘ │
└─────────────────────────────┘
```

## 4. 枠オーバーレイ仕様

### 4.1 形状と描画

- ウィンドウリージョンで中央を刳り抜いた「縁の帯 + 上辺タグ」だけの形状。内側には何も被さらず、領域内のアプリはクリック・入力・スクロールとも通常どおり操作できる。
- 枠線の見た目は 2〜3px。**枠線は領域の外側に配置**する(内側だと当たり判定が共有領域の縁を覆い、縁にある UI が操作不能になるため。また「線の内側に見えるもの = 視聴者に見えるもの」の WYSIWYG が成立する)。
- 領域の辺がモニタ端に接する場合、その辺の枠線は画面外に切れてよい(割り切り。画面端は境界として自明であり、外方向へのリサイズも不要なため)。
- 枠・タグ・ハンドル・解像度バッジには `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)` を設定し、キャプチャ(= 共有映像)から除外する。

### 4.2 リサイズ

- 当たり判定は 6px 幅(見た目より広い。`WM_NCHITTEST` の判定域は見た目と独立に設定)。
- `WM_NCHITTEST` で辺・角に応じ `HTLEFT` / `HTTOPRIGHT` 等を返し、OS 標準のリサイズドラッグに乗る(カーソル形状の変化も OS 任せ)。
- リサイズ中は解像度バッジ(例: 1280 × 720)をリアルタイム表示。
- Shift 押下でアスペクト比固定。キリのいい解像度(1280×720、1920×1080 等)へのスナップは検討事項。
- 最小サイズを設ける(値は未定)。

### 4.3 移動(タグ / アイランド)

- 上辺に小さなタグを**内側にはみ出す**形で表示する。
- タグを掴んでドラッグすると領域全体が移動する(`WM_NCHITTEST` で `HTCAPTION` を返し、OS 標準の移動ドラッグに乗る)。
- タグ上の「||」グリップをドラッグすると、タグ自体を上辺に沿って横スライドできる(タグの裏に操作したいものがある場合の逃がし)。
- タグの裏は例外的に操作不能領域となるが、横スライドで回避可能とする。

### 4.4 コンテキストメニュー

タグの右クリックでメニューを表示する。項目(案):

- 共有の一時停止 / 再開
- 領域プリセットの保存 / 呼び出し
- アスペクト比固定の切り替え
- 終了

### 4.5 ドラッグ・リサイズ中の映像

移動・リサイズ中はミラーウィンドウの映像を**追従させない**。直前フレームを固定表示し、確定時(マウスアップ)に新しい領域で更新する。

## 5. ミラーウィンドウ仕様

- 通常のトップレベルウィンドウ(タイトルバーあり、Alt+Tab に出る)。Teams の共有ピッカーに「ウィンドウ」として列挙される。
- **キャプチャ除外は設定しない**(設定すると Teams から真っ黒になる)。除外するのは枠オーバーレイのみ。
- タイトルは「ClipFrame View」。専用アイコンを付け、共有ピッカーで見つけやすくする。
- 起動時に共有領域の**外**へ自動配置する。ドラッグで領域と重なった場合は警告(自己映り込み = 合わせ鏡の防止)。
- **表示状態と共有可否**: Teams のウィンドウ共有(Windows.Graphics.Capture)は DWM の持つウィンドウ自身の描画面を取得するため、表示状態の影響は次のとおり。
  - 他ウィンドウの裏に隠れている: 問題なし(重なり順は無関係)。
  - 一部が画面外にはみ出している: 基本的に問題なし(モニタ表示範囲に依存しない)。※プロトタイプで実機確認すること。
  - 最小化: **不可**。DWM が描画を止め、Teams 側の映像が最後のフレームで凍結する。
- 最小化事故の防止: `SC_MINIMIZE` をフックし、最小化の代わりに「画面隅への退避」に置き換える。
- 領域のリサイズに応じてミラーウィンドウのアスペクト比も追従する。

## 6. キャプチャ仕様

- API: Windows.Graphics.Capture(WinRT)。`IGraphicsCaptureItemInterop::CreateForMonitor` で対象モニタをキャプチャし、指定領域をクロップして表示する。
- フレームレート: 30fps 程度を目安とし、負荷とのバランスで調整。
- OS がキャプチャ対象(モニタ全体)に表示する黄色い枠はそのまま許容する(「キャプチャ中」の明示としてむしろ歓迎)。共有領域の明示は自前の枠オーバーレイが担う。

## 7. 技術仕様

| 項目 | 内容 |
|---|---|
| 言語 / UI | C# / WPF |
| ランタイム | .NET 10(LTS、サポート 2028-11 まで) |
| TFM | `net10.0-windows10.0.19041.0`(WinRT API が標準投影される) |
| 最低 OS | Windows 10 2004(`SupportedOSPlatformVersion` 10.0.19041。`WDA_EXCLUDEFROMCAPTURE` の要件) |
| 描画方式 | 第一段階: CPU コピー + `WriteableBitmap`(フル HD 30fps 想定)。負荷が問題になれば `HwndHost` + DXGI スワップチェーンへ差し替え |
| 配布 | `PublishSingleFile` + self-contained の単一 EXE(ランタイム同梱) |

主な Win32 相互運用: `SetWindowDisplayAffinity`、`WM_NCHITTEST`、`SetWindowRgn`(ウィンドウリージョン)、`IGraphicsCaptureItemInterop`。

## 8. スコープ外・割り切り

- 共有機能そのものは持たない(Teams 等の会議アプリに委ねる)。
- シングルモニタで領域を画面ほぼ全体にするケースは対象外(ミラーウィンドウの置き場がない。その場合は通常の画面全体共有で足りる)。
- 静止画スクリーンショット機能は持たない(Win+Shift+S で足りる)。
