# ClipFrame(画面部分切り取り君II)

画面の**任意領域だけ**を、Teams などのウィンドウ共有で共有できるようにする Windows アプリです。

Microsoft Teams の画面共有は「画面全体」か「ウィンドウ」単位しか選べず、Zoom のような
「画面の一部」共有ができません。ClipFrame は指定した領域をリアルタイムに映し続ける
**通常のウィンドウ(ミラーウィンドウ)** を用意します。Teams からはそれが普通のウィンドウとして
見えるので、ウィンドウ共有を使えば実質的に「領域共有」が実現できます。

> 本アプリ自体は共有機能を持ちません。「領域を指定し、その内容をウィンドウとして見えるようにする」
> ことに専念し、共有は Teams 等の会議アプリに委ねます。

---

## 仕組み

2 つのウィンドウで構成されます。

```
デスクトップ
┌─────────────────────────────┐
│  ┌──[⣿ ClipFrame  1280×720]─┐ │
│  │                          │ │   ① 枠オーバーレイ
│  │   共有したい領域          │ │      ・領域を可視化/操作する枠
│  │   (素通し・操作できる)   │ │      ・キャプチャからは除外され共有映像に映らない
│  └──────────────────────────┘ │
│                  ┌──────────┐  │
│                  │ClipFrame │  │   ② ミラーウィンドウ
│                  │  View    │  │      ・領域の中身を映す普通のウィンドウ
│                  └──────────┘  │      ・Teams でこれを共有する
└─────────────────────────────┘
```

- **枠オーバーレイ** … 共有領域を示す枠。中央はくり抜かれており、枠の内側にあるアプリは
  そのままクリック・入力・スクロールできます。枠自身は `WDA_EXCLUDEFROMCAPTURE` で
  キャプチャから除外されるため、共有映像には枠線もタグも映りません。
- **ミラーウィンドウ** … 領域の中身をリアルタイム表示する通常ウィンドウ。Teams の共有ピッカーに
  「ClipFrame View」として並ぶので、これを選んで共有します。

---

## 主な機能

| 分類 | 内容 |
|---|---|
| 領域の移動 | 上辺のタグをドラッグ。タグの「グリップ」を掴めばタグ自体を左右へスライド(裏のUIを逃がす) |
| 領域のリサイズ | 枠の辺・角をドラッグ。当たり判定は広めに設定 |
| 解像度スナップ | 1280×720・1920×1080 等へ吸着。**Alt** 押下で一時無効、メニューでON/OFF |
| アスペクト比固定 | **Shift** を押しながらリサイズ |
| 解像度バッジ | リサイズ中に現在の解像度(例: 1280 × 720)をタグに表示 |
| 一時停止/再開 | 共有映像を直前フレームで止める/再開する |
| 領域プリセット | 現在の領域に名前を付けて保存・呼び出し・削除(`%APPDATA%\ClipFrame\presets.json`) |
| フレームレート | 既定 15fps。メニューで 10/15/20/30 fps を選択(負荷とのバランス調整) |
| 最小化事故の防止 | ミラーを最小化しようとすると画面隅へ退避(最小化すると共有映像が固まるため) |
| 自己映り込み防止 | ミラーが共有領域に重なると警告(合わせ鏡の防止) |

操作メニューは**タグの右クリック**から開きます。

---

## 使い方

1. `ClipFrame.exe` を起動すると、枠オーバーレイとミラーウィンドウ(ClipFrame View)が現れます。
2. 枠をドラッグ/リサイズして、共有したい領域に合わせます。
3. Teams の画面共有で「ウィンドウ」→「**ClipFrame View**」を選択します。
4. 枠内のアプリを普段どおり操作すると、その領域だけが相手に共有されます。

> モニタ全体に黄色い「キャプチャ中」枠が出ますが、これは OS の仕様です(共有中の明示として許容)。

## 動作要件

- Windows 10 バージョン 2004(build 19041)以降 / Windows 11
- 単一 EXE 版(self-contained)はランタイム同梱のため .NET のインストール不要

## ビルドと実行

開発には **.NET 10 SDK** が必要です。

```powershell
# 実行
cd src/ClipFrame
dotnet run -c Debug

# 配布用の単一 EXE(ランタイム同梱)
dotnet publish -c Release
# 出力: src/ClipFrame/bin/Release/net10.0-windows10.0.19041.0/win-x64/publish/ClipFrame.exe
```

Visual Studio で開く場合はソリューション `src/ClipFrame.slnx` を使用します。

技術スタック: C# / WPF / .NET 10、キャプチャは Windows.Graphics.Capture(WGC)、
D3D11 相互運用に Vortice.Windows。

## 現時点の割り切り

- 共有機能そのものは持ちません(Teams 等に委ねる)。
- ホットキーは未実装。
- 領域は単一モニタ内を想定(複数モニタをまたぐ領域は領域中心のモニタを対象にクロップ)。
- シングルモニタで領域を画面ほぼ全体にするケースは対象外(ミラーの置き場がないため。
  その場合は通常の画面全体共有で足ります)。
